## 一、OpenClaw 是什么

OpenClaw 是一个开源的 AI 自动化执行平台。

简单来说，它像是一个**智能遥控器**——你可以通过自然语言指令，让 AI 帮你执行各种任务：查天气、发消息、操作浏览器、读写文件、甚至管理你的代码仓库。

不同于单一的 AI 聊天工具，OpenClaw 的核心价值在于**执行**。

它把 AI 的"理解能力"与系统的"操作能力"连接起来，让 AI 从"只能说话"进化到"能做事"。

## 二、安装前的准备

在动手安装之前，先问自己三个问题：

### 🤔 问题1：你要用 OpenClaw 做什么？

- **如果只是**想聊天、生成图片——**没必要安装**。用 ChatGPT、Claude、Kimi 这些现成的工具就够了。
- **如果是**想打造自动化工作流、系统性地构建自己的 AI 助手，或者单纯想动手体验——**可以安装**。

### 💰 问题2：你每月打算花多少钱？

不同需求有不同的成本底线：

| 方案 | 月成本 | 说明 |
|------|--------|------|
| **最低体验版** | ~40元 | 字节 Coze Coding Plan Lite，适合入门体验 |
| **自运维版** | 100-300元+ | 云服务器费用 + 模型订阅（Kimi/Minimax 等） |
| **本地部署版** | 模型费用 | 无服务器成本，但需要硬件支持 |

**建议**：先试试 40 元的 Coze Lite 方案，确认真的有需求再升级。

### 🔧 问题3：你是技术人员还是非技术人员？

| 你的情况 | 推荐方案 | 原因 |
|----------|----------|------|
| **完全无技术背景** | 用封装好的平台（国内推荐字节 Coze） | 开箱即用，省心省力 |
| **有一定技术能力** | 建议自部署 OpenClaw | 灵活可控，数据自主 |

**技术人员特别提醒**：
- 建议选择**国区之外的云服务器**部署，可获取的信息资源更丰富
- 如果你的需求都在国区内，用封装平台反而更简单
- 国外用户推荐 Google 生态，整体工具链成熟

---

想清楚了以上三个问题，确认 OpenClaw 适合你，再继续看下面的环境要求：

### 2.1 系统要求

| 平台 | 支持情况 | 备注 |
|------|----------|------|
| Linux | ✅ 完全支持 | 推荐 Ubuntu 20.04+ / Debian 11+ |
| macOS | ✅ 支持 | 推荐 macOS 13+ |
| Windows | ⚠️ 需 WSL2 | 通过 Windows Subsystem for Linux |

### 2.2 依赖环境

OpenClaw 基于 Node.js 构建，需要：

- **Node.js**: 22.x 或更高版本
- **npm**: 10.x 或更高版本
- **Git**: 用于克隆仓库和更新

检查你的 Node.js 版本：

```bash
node --version  # 应显示 v22.x.x 或更高
npm --version   # 应显示 10.x.x 或更高
```

如果版本过低，建议先升级 Node.js（可以使用 nvm）：

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.0/install.sh | bash
source ~/.bashrc
nvm install 22
nvm use 22
```

## 三、三种部署方案详解

在正式安装之前，你需要选择一种部署方式。OpenClaw 有三种主流部署方案，各有优劣：

### 3.1 云端自部署（云养虾） ☁️

**定义**：在租用的云服务器上自行部署 OpenClaw。

**适合人群**：有一定技术基础，希望灵活可控的用户

**优点**：
- ✅ **数据自主**：数据存储在自己的服务器上
- ✅ **灵活可控**：可以自由选择服务器位置（国区/海外）
- ✅ **访问稳定**：24小时在线，不依赖本地网络
- ✅ **可扩展**：可根据需求随时升级配置

**缺点**：
- ❌ **需要运维能力**：需要自己维护服务器
- ❌ **有服务器成本**：月费约 50-200 元（取决于配置和位置）
- ❌ **需要域名/备案**（国区服务器）：如果选国区，需要备案

**推荐服务商**：
| 服务商 | 价格区间 | 特点 | 推荐场景 |
|--------|----------|------|----------|
| **阿里云 ECS** | ￥60-200/月 | 国内访问快，需备案 | 国内用户，访问国内资源 |
| **腾讯云 CVM** | ￥60-200/月 | 国内访问快，需备案 | 国内用户，访问国内资源 |
| **Vultr** | $5-20/月 | 海外节点多，免备案 | 需要访问海外资源 |
| **DigitalOcean** | $6-24/月 | 简单易用，文档丰富 | 新手海外部署 |

**部署建议**：
- 最低配置：1核 2GB 内存（可运行基础功能）
- 推荐配置：2核 4GB 内存（流畅运行 + 多 Agent）
- 系统：Ubuntu 22.04 LTS

---

### 3.2 托管服务（托管虾） 🏨

**定义**：使用第三方托管平台，无需自己维护服务器。

**适合人群**：无技术背景，或不想折腾服务器的用户

**优点**：
- ✅ **开箱即用**：注册即用，无需配置环境
- ✅ **零运维**：平台负责维护和更新
- ✅ **成本可控**：按需付费，起步成本低
- ✅ **自带生态**：通常集成了模型、插件等

**缺点**：
- ❌ **数据在第三方**：隐私性相对较弱
- ❌ **功能受限**：只能用平台提供的功能，难以深度定制
- ❌ **服务器位置固定**：无法自由选择部署位置
- ❌ **长期成本可能更高**：使用量大时费用会累积

**推荐平台**：
| 平台 | 月成本 | 特点 | 适合人群 |
|------|--------|------|----------|
| **字节 Coze** | ￥40+/月 | 国内生态完善，插件丰富 | 国内用户，快速上手 |
| **Dify** | 免费/付费 | 开源可自托管，可视化工作流 | 需要工作流编排 |
| **n8n Cloud** | $20+/月 | 国外老牌自动化平台 | 海外用户，英文友好 |

**重要提醒**：
托管平台本质上是用**易用性**换**自由度**。如果你只是想快速体验 AI 自动化，托管平台是好选择；但如果你希望深度定制、数据自主，建议考虑自部署。

---

### 3.3 本地部署（本地养虾） 🏠

**定义**：在自己的电脑或 NAS 上部署 OpenClaw。

**适合人群**：注重隐私、有闲置设备的技术爱好者

**优点**：
- ✅ **完全隐私**：数据不出本机，最安全
- ✅ **零服务器成本**：利用现有设备
- ✅ **局域网高速**：本地访问速度极快
- ✅ **完全控制**：想怎么改就怎么改

**缺点**：
- ❌ **设备必须开机**：电脑/NAS 关机就无法访问
- ❌ **外网访问麻烦**：需要内网穿透或公网 IP
- ❌ **性能受限**：取决于你的设备配置
- ❌ **需要一定技术**：环境配置、网络配置等

**推荐场景**：
- 有一台 24 小时开机的 NAS（群晖、威联通等）
- 有一台闲置的笔记本/树莓派
- 主要在局域网内使用（家庭/办公室自动化）

**硬件建议**：
| 设备类型 | 最低配置 | 推荐配置 | 备注 |
|----------|----------|----------|------|
| **旧电脑** | 4核 4GB | 4核 8GB | Windows/Mac/Linux 均可 |
| **树莓派 4B** | 4GB 版本 | 8GB 版本 | 需要散热，性能一般 |
| **NAS** | 4核 4GB | 4核 8GB+ | 群晖/威联通需开启 SSH |

**外网访问方案**：
1. **Tailscale**（推荐）：免费，简单，无需公网 IP
2. **Cloudflare Tunnel**：免费，需配置
3. **frp 内网穿透**：需一台云服务器做中转

---

### 三种方案对比总结

| 维度 | 云端自部署 ☁️ | 托管服务 🏨 | 本地部署 🏠 |
|------|----------------|--------------|--------------|
| **技术门槛** | 中等 | 低 | 中高 |
| **月成本** | ￥50-200 | ￥40-200+ | ￥0（电费和设备除外） |
| **数据隐私** | 高 | 中 | 最高 |
| **可定制性** | 高 | 低 | 最高 |
| **24小时在线** | ✅ | ✅ | 需设备开机 |
| **外网访问** | 直接访问 | 直接访问 | 需额外配置 |
| **推荐人群** | 技术人员 | 非技术/快速体验 | 隐私敏感/有闲置设备 |

**选择建议**：
- **新手想快速体验** → 选**托管服务**（字节 Coze）
- **技术人员长期使用** → 选**云端自部署**（海外 VPS）
- **注重隐私/有 NAS** → 选**本地部署**

选定方案后，继续看下面的具体安装步骤：

## 四、安装方式详解（按部署方案）

根据你选择的部署方案，选择对应的安装方式。

### 4.1 NPM 全局安装（云端/本地部署推荐）

适合**云端自部署**和**本地部署**用户，一键安装，自动更新。

```bash
# 安装 OpenClaw CLI
npm install -g openclaw

# 验证安装
openclaw --version
```

安装完成后，运行初始化命令：

```bash
openclaw setup
```

这个命令会：
- 创建默认配置目录（`~/.openclaw/`）
- 检查系统依赖
- 引导你完成基础配置

**适用场景**：
- ✅ 云服务器部署
- ✅ 本地电脑部署
- ✅ NAS 部署

**优点**：简单快速，适合新手
**缺点**：版本受限于 npm 仓库更新频率

---

### 4.2 源码安装（开发者/深度定制）

适合想要**深度定制**或参与开发的用户。

```bash
# 克隆仓库
git clone https://github.com/openclaw/openclaw.git
cd openclaw

# 安装依赖
npm install

# 构建项目
npm run build

# 链接到全局（可选，便于命令行调用）
npm link
```

源码安装后，你可以：
- 修改核心代码
- 开发自定义插件
- 参与社区贡献

**适用场景**：
- ✅ 需要修改核心功能
- ✅ 开发自定义插件
- ✅ 参与 OpenClaw 开源贡献

**优点**：完全可控，可定制
**缺点**：需要手动更新，维护成本较高

---

### 4.3 Docker 安装（快速体验/隔离环境）

适合想快速体验或隔离环境的用户。

```bash
# 拉取镜像
docker pull openclaw/openclaw:latest

# 运行容器
docker run -it \
  -v ~/.openclaw:/root/.openclaw \
  -v $(pwd):/workspace \
  openclaw/openclaw:latest
```

**优点**：环境隔离，一键启动
**缺点**：部分系统级功能受限

## 六、初始化与配置

安装完成后，需要进行初始配置。

### 6.1 运行配置向导

```bash
openclaw setup
```

向导会引导你完成：
1. 选择默认 AI 模型
2. 配置消息渠道（飞书/微信等）
3. 设置工作目录
4. 启用/禁用插件

### 6.2 手动配置文件

高级用户可以手动编辑配置文件：

```bash
# 打开配置文件
nano ~/.openclaw/config.yaml
```

关键配置项：

```yaml
# AI 模型配置
model:
  default: kimi-coding/kimi-for-coding
  fallback: openai/gpt-4

# 消息渠道配置
channels:
  feishu:
    enabled: true
    webhook_url: your-webhook-url
  
# 插件配置
plugins:
  browser:
    enabled: true
  healthcheck:
    enabled: true
```

### 6.3 安装推荐插件

```bash
# 查看可用插件
openclaw plugin list

# 安装浏览器插件
openclaw plugin install browser

# 安装健康检查插件
openclaw plugin install healthcheck

# 安装 Feishu 插件
openclaw plugin install feishu
```

## 七、常见问题与解决方案

### Q1: 安装时提示权限不足？

**现象**：`EACCES: permission denied`

**解决**：

```bash
# 方法1：修改 npm 全局目录权限
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc

# 方法2：使用 sudo（不推荐长期使用）
sudo npm install -g openclaw
```

### Q2: 如何切换不同的 AI 模型？

```bash
# 临时切换
openclaw --model openai/gpt-4

# 永久修改默认模型
openclaw config set model.default openai/gpt-4
```

### Q3: 更新后配置丢失？

OpenClaw 的配置存储在 `~/.openclaw/` 目录，正常情况下更新不会丢失。如遇到：

```bash
# 备份配置
cp -r ~/.openclaw ~/.openclaw.backup

# 重新安装后恢复
cp -r ~/.openclaw.backup/* ~/.openclaw/
```

### Q4: 如何完全卸载？

```bash
# NPM 方式
npm uninstall -g openclaw
rm -rf ~/.openclaw

# 源码方式
cd /path/to/openclaw
npm unlink
rm -rf /path/to/openclaw
```
