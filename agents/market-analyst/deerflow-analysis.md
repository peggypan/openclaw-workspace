## 一、项目概述

DeerFlow（**D**eep **E**xploration and **E**fficient **R**esearch **F**low）是字节跳动开源的超级代理框架，基于 LangGraph 和 LangChain 构建。

### 核心定位
- 最初是 Deep Research（深度研究）框架
- 2.0 版本彻底重写，升级为「代理运行时基础设施」
- 不只是聊天机器人，而是给 AI 代理提供完整的「工作电脑」

### 技术栈
- **后端**: Python (55.9%) - LangGraph Agent 运行时
- **前端**: TypeScript (26.3%) - React + 可视化界面
- **其他**: HTML, CSS, JavaScript, Shell

## 二、整体架构

```
┌─────────────────────────────────────────────────────────┐
│                    前端 (TypeScript)                      │
│              React + 可视化工作流界面                      │
└─────────────────────────────────────────────────────────┘
                           │
┌─────────────────────────────────────────────────────────┐
│                   Gateway API 层                         │
│         统一入口：技能管理 / 文件上传 / 渠道接入           │
│         HTTP API + WebSocket 实时推送                    │
└─────────────────────────────────────────────────────────┘
                           │
┌─────────────────────────────────────────────────────────┐
│              LangGraph Agent 运行时                      │
│    ┌─────────────┐    ┌─────────────┐    ┌──────────┐  │
│    │  Lead Agent │───→│ Sub-Agent 1 │    │ Sub-A... │  │
│    │   (主代理)   │←───│             │    │          │  │
│    └─────────────┘    └─────────────┘    └──────────┘  │
│           │                                              │
│    ┌─────────────┐    ┌─────────────┐    ┌──────────┐  │
│    │   Skills    │    │   Memory    │    │ Sandbox  │  │
│    │   (技能)    │    │   (记忆)    │    │ (沙盒)   │  │
│    └─────────────┘    └─────────────┘    └──────────┘  │
└─────────────────────────────────────────────────────────┘
```

### 数据流向
1. 用户请求 → Gateway 接收
2. Gateway 创建线程，调用 LangGraph Server
3. Lead Agent 分析任务，决定是自己执行还是拆分子代理
4. 执行过程中读写 Memory、调用 Skills、在 Sandbox 中运行代码
5. 结果流式返回给用户

## 三、子代理调度机制

### 3.1 任务拆解流程示例

**用户输入**: "帮我调研 AI 编程工具市场，输出一份 PPT"

```
        ↓
┌─────────────────┐
│   Lead Agent    │ ← 分析任务复杂度，决定拆解策略
│   (规划阶段)     │
└─────────────────┘
        ↓
┌──────────────────────────────────────────────────┐
│  拆解为并行子任务：                                │
│  1. 搜索当前主流 AI 编程工具 (Cursor, Windsurf等)  │
│  2. 分析市场规模和增长数据                          │
│  3. 调研用户痛点和评价                              │
│  4. 整理技术趋势和竞品对比                          │
└──────────────────────────────────────────────────┘
        ↓
┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐
│ Sub-A 1 │ │ Sub-A 2 │ │ Sub-A 3 │ │ Sub-A 4 │
│ (工具)  │ │ (市场)  │ │ (用户)  │ │ (趋势)  │
└─────────┘ └─────────┘ └─────────┘ └─────────┘
     ↓           ↓           ↓           ↓
   独立执行    独立执行     独立执行     独立执行
   (可并行)    (可并行)     (可并行)     (可并行)
     ↓           ↓           ↓           ↓
┌──────────────────────────────────────────────────┐
│   Lead Agent                                     │
│   (综合阶段) ← 汇总结果，去重、整合、生成大纲       │
└──────────────────────────────────────────────────┘
        ↓
┌──────────────────────────────────────────────────┐
│  第二阶段子代理：                                   │
│  5. 根据大纲生成 PPT 内容 (每页一个子代理)          │
│  6. 搜索/生成配图                                   │
└──────────────────────────────────────────────────┘
        ↓
   最终 PPT 输出
```

### 3.2 子代理隔离机制

| 隔离维度 | 说明 |
|---------|------|
| **上下文隔离** | 子代理看不到主代理或其他子代理的对话历史 |
| **工具隔离** | 可分配不同的技能集给不同子代理 |
| **终止条件** | 每个子代理有独立的终止判断（完成标志、token 限制） |
| **结果回传** | 结构化输出（JSON），主代理按字段解析 |

### 3.3 调度优化
- **并行度控制**: 根据任务类型和资源限制调整并发数
- **依赖管理**: 子任务之间有依赖时自动串行化
- **超时处理**: 单个子代理超时不会阻塞整个任务
- **失败重试**: 支持子代理级别重试策略

## 四、技能系统（Skills）

Skills 是 DeerFlow 的**能力扩展单元**。

### 4.1 Skill 目录结构

```
skills/research/
├── SKILL.md           # 核心定义（必需）
├── examples/          # 示例输入输出
├── templates/         # 提示词模板
└── tools/             # 自定义工具脚本
```

### 4.2 SKILL.md 示例

```markdown
# Research Skill

## 能力
执行深度网络搜索，多源验证，生成结构化研究报告

## 输入
- topic: 研究主题
- depth: 研究深度 (quick/standard/deep)
- sources: 偏好的数据源

## 输出
- report: Markdown 格式报告
- sources: 引用来源列表
- confidence: 置信度评分

## 工作流
1. 使用 web_search 工具收集信息
2. 交叉验证多个来源
3. 使用 report-generation 技能生成最终报告

## 工具依赖
- web_search
- web_fetch
- file_write
```

### 4.3 技能加载机制

**懒加载（Lazy Loading）**:
- 不是启动时加载所有技能
- Lead Agent 分析任务 → 确定需要的技能 → 动态加载
- 保持上下文窗口精简

**技能路由示例**:
```
任务: "生成一份关于新能源汽车的PPT"

Lead Agent 判断需要的技能：
├── research (调研主题)
├── report-generation (整理内容)
├── slide-creation (生成PPT结构)
└── image-generation (生成配图)
```

### 4.4 MCP Server 集成

DeerFlow 支持通过 **MCP (Model Context Protocol)** 接入外部工具：

```
┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│   DeerFlow  │ ───→ │  MCP Server │ ───→ │  External   │
│             │ ←─── │  (HTTP/SSE) │ ←─── │   Tools     │
└─────────────┘      └─────────────┘      └─────────────┘
                            │
                    ┌───────┴───────┐
                    ↓               ↓
              ┌─────────┐     ┌─────────┐
              │Database │     │  APIs   │
              │ Search  │     │ (GitHub │
              │         │     │  Slack) │
              └─────────┘     └─────────┘
```

支持 OAuth 认证流程（client_credentials、refresh_token）

## 五、沙盒执行环境

DeerFlow 区别于普通 ChatBot 的关键——**代理拥有真实的执行环境**。

### 5.1 沙盒架构

```
┌────────────────────────────────────────────┐
│              Host Machine                  │
│  ┌──────────────────────────────────────┐  │
│  │         DeerFlow Backend             │  │
│  └──────────────────────────────────────┘  │
│                    │                       │
│  ┌──────────────────────────────────────┐  │
│  │      Docker Daemon / Kubernetes      │  │
│  └──────────────────────────────────────┘  │
│         │              │        │          │
│    ┌────┘         ┌────┘   ┌───┘          │
│    ↓              ↓        ↓              │
│ ┌──────┐      ┌──────┐  ┌──────┐          │
│ │Task 1│      │Task 2│  │Task N│          │
│ │容器  │      │容器  │  │容器  │          │
│ └──┬───┘      └──┬───┘  └──┬───┘          │
└────┼─────────────┼─────────┼──────────────┘
     │             │         │
     ↓             ↓         ↓
┌────────────────────────────────────────────┐
│  /mnt/user-data/                           │
│  ├── uploads/     ← 用户上传文件            │
│  ├── workspace/   ← 代理工作目录            │
│  └── outputs/     ← 最终产出物              │
│                                            │
│  /mnt/skills/                              │
│  ├── public/      ← 内置技能               │
│  └── custom/      ← 自定义技能             │
└────────────────────────────────────────────┘
```

### 5.2 三种执行模式

| 模式 | 适用场景 | 隔离性 | 性能 |
|------|----------|--------|------|
| **Local** | 本地开发、测试 | 无隔离 | 最高 |
| **Docker** | 生产环境 | 容器隔离 | 中等 |
| **Kubernetes** | 大规模部署 | Pod 隔离 | 按需扩展 |

### 5.3 沙盒内能力

代理在沙盒中可以：
- **文件操作**: 读写、编辑、创建目录
- **代码执行**: 运行 Python、Bash、Node.js 等
- **网络访问**: 访问互联网（可配置限制）
- **图像处理**: 查看、分析、生成图片
- **安装依赖**: pip install、npm install 等

**安全边界**:
- 容器资源限制（CPU、内存）
- 网络策略（egress 限制）
- 文件系统只读/读写权限控制
- 超时强制终止

## 六、记忆系统

DeerFlow 的**长期记忆**让代理能"越用越懂你"。

### 6.1 记忆层级

```
┌─────────────────────────────────────────────┐
│  Layer 3: Knowledge Base (知识库)            │
│  ├── 领域知识（如用户的行业背景）              │
│  ├── 偏好规则（如输出格式要求）                │
│  └── 常用工作流模板                           │
└─────────────────────────────────────────────┘
                    ↑ 累积
┌─────────────────────────────────────────────┐
│  Layer 2: Session Memory (会话记忆)          │
│  ├── 当前对话历史                             │
│  ├── 中间结果摘要                             │
│  └── 临时文件引用                             │
└─────────────────────────────────────────────┘
                    ↑ 沉淀
┌─────────────────────────────────────────────┐
│  Layer 1: Working Memory (工作记忆)          │
│  └── 当前上下文窗口中的内容                    │
└─────────────────────────────────────────────┘
```

### 6.2 记忆存储结构

**本地存储**: `~/.deerflow/memory/`

```
memory/
├── profiles/
│   └── user_xxx.json      # 用户画像
├── sessions/
│   ├── session_001/       # 会话数据
│   │   ├── messages.jsonl
│   │   ├── summaries/
│   │   └── artifacts/
│   └── session_002/
└── facts/
    ├── preferences.json   # 提取的偏好
    └── facts.jsonl        # 事实知识库
```

### 6.3 记忆更新机制

**自动提取**:
```
对话内容 → LLM 分析 → 提取关键信息 → 去重检查 → 写入记忆

示例：
输入："我喜欢简洁的汇报，不要超过3页"
提取：{"preference": "report_length", "value": "max_3_pages", "confidence": 0.95}
```

**去重策略**:
- 语义相似度检测
- 冲突时保留高置信度/新数据
- 定期合并重复条目

### 6.4 记忆召回

查询时按优先级召回：
1. 当前会话相关上下文（最高优先级）
2. 用户画像匹配项
3. 历史相似任务的经验

## 七、部署实践

### 7.1 最小部署配置

```yaml
# config.yaml
models:
  - name: deepseek-chat
    display_name: DeepSeek V3
    use: langchain_openai:ChatOpenAI
    model: deepseek-chat
    api_key: $DEEPSEEK_API_KEY
    base_url: https://api.deepseek.com/v1
    max_tokens: 8192

sandbox:
  use: deerflow.community.local_sandbox:LocalSandboxProvider
  # 生产环境换成 DockerSandboxProvider

channels:
  feishu:
    enabled: true
    app_id: $FEISHU_APP_ID
    app_secret: $FEISHU_APP_SECRET
```

### 7.2 启动流程

```bash
# 1. 初始化配置
make config

# 2. 编辑 .env 填入 API Keys

# 3. Docker 启动（推荐）
make docker-init    # 拉取沙盒镜像
make docker-start   # 启动所有服务

# 4. 访问
open http://localhost:2026
```

### 7.3 支持的多渠道接入

| 渠道 | 传输方式 | 难度 |

| **Telegram** | Bot API (长轮询) | 简单 |
| **Slack** | Socket Mode | 中等 |
| **飞书/Lark** | WebSocket | 中等 |

| 维度 | 传统 ChatBot | DeerFlow |
|------|-------------|----------|

如果你要做一个真正能干活的 AI 助手（而不是只会聊天的），DeerFlow 是目前开源方案中最完整的框架之一。
